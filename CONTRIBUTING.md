# Guia de Contribuição

Obrigado por considerar contribuir com o PortfolioCSH!

## Como Contribuir

1. **Reporte Problemas**
   - Verifique se o problema já não foi reportado
   - Inclua detalhes e etapas para reproduzir

2. **Sugira Melhorias**
   - Descreva claramente sua sugestão
   - Explique o benefício da mudança

3. **Envie Pull Requests**
   - Siga o padrão de código existente
   - Inclua testes quando aplicável
   - Documente suas mudanças

## Padrões de Código
- Use nomes descritivos para variáveis e métodos
- Comente código complexo
- Mantenha o estilo consistente

## Processo
1. Discuta grandes mudanças via Issues primeiro
2. Mantenha os commits atômicos e bem descritos
3. Atualize a documentação conforme necessário
